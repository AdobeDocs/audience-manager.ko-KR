# 데이터 피드 만들기, 가격 및 관리 {#create-price-and-manage-data-feeds}

## 공개 또는 비공개 데이터 피드 만들기 {#create-public-private-data-feed}

데이터 피드에는 이름, 설명, 데이터 소스 및 계획 유형이 필요합니다. 피드를 저장하고 활성화할 때까지 피드가 비활성화됩니다. 에서 공개 또는 비공개 데이터 피드 설정 **[!UICONTROL Audience Marketplace > My Shared Data]**. 데이터 판매자만 사용할 수 있습니다.

<!-- t_data_feed.xml -->

공개 또는 비공개 데이터 피드를 만들려면 관리자 권한이 있어야 합니다.
데이터 피드를 만들려면 다음 작업을 수행하십시오.

1. 클릭 **[!UICONTROL New Data Feed]**.
1. 데이터 피드에 이름을 지정합니다. 데이터 구매자는 이름을 기반으로 피드를 검색할 수 있습니다.
1. 간단한 설명(최대 255자)을 제공합니다.

   설명을 잘 적으면 피드를 정확하게 설명할 수 있습니다. 예를 들어 마케팅 카테고리, 인구 통계 및 지리적 적용 범위에 대한 텍스트를 포함할 수 있습니다(예: [!DNL US] 또는 북미). 설명 텍스트는 검색할 수 있으며 구매자가 피드를 찾거나 평가할 수 있도록 도와줍니다. 좋은 설명은 데이터 피드에 가입자를 유도하는 데 중요한 부분입니다.
1. 에서 데이터 소스 선택 **[!UICONTROL Data Source]** 옵션. 데이터 피드는 단일 데이터 소스로 제한됩니다. 동일한 데이터 피드에 여러 데이터 소스를 할당할 수 없습니다.

   >[!IMPORTANT]
   >
   >이 데이터 소스에 속하는 현재 및 향후 모든 트레이트는 이 피드의 일부로 데이터 구매자와 공유됩니다.

1. 위치 [!UICONTROL Plan Types]을 클릭하고 사용할 옵션을 선택한 다음 을 클릭합니다. **[!UICONTROL Add Plan]**.

   피드는 여러 계획을 포함할 수 있습니다. 플랜에는 여러 사용 사례가 포함될 수 있습니다. 자세한 내용은 [데이터 피드에 대한 계획 유형](../../../features/audience-marketplace/marketplace-data-providers/marketplace-create-manage-feeds.md#plan-types).

1. 클릭 **[!UICONTROL Save]** 데이터 피드를 저장하려면 *없이* 활성화.
1. 데이터 피드를 저장하고 활성화하려면 다음 작업을 수행하십시오.
   1. 이동 **[!UICONTROL Availability]** 슬라이더 **[!UICONTROL Active]**.
   1. 클릭 **[!UICONTROL Save]**.

   >[!NOTE]
   >
   >* 저장되고 활성화된 데이터 피드는 삭제할 수 없습니다.
   >* 구매자는 활성 피드만 봅니다.


### 선택 사항: 개인 데이터 피드 만들기

다음에서 [!UICONTROL Settings] 섹션, 슬라이더를 다음으로 이동:

* **[!UICONTROL Private]** 및 **[!UICONTROL Branded]**: 구매자 [!UICONTROL Marketplace] list는 공급자 열에 판매자 이름을 표시하며 다른 모든 데이터는 숨겨집니다.

* **[!UICONTROL Private]** 및 **[!UICONTROL Unbranded]**: 구매자 [!UICONTROL Marketplace] 목록에는 데이터 피드 이름과 설명만 표시됩니다. 데이터 공급자 이름은 로 표시됩니다. [!UICONTROL Private Seller].

비공개 피드가 구매자에게 어떻게 보이는지 확인하려면 의 구매자 섹션을 참조하십시오. [비공개 데이터 피드](../../../features/audience-marketplace/marketplace-private-feeds.md).

## 가입자의 데이터 피드 비활성화 {#deactivate-data-feed}

(으)로 [!UICONTROL Audience Marketplace] 데이터 공급자는 구독한 데이터 피드에 대한 구매자 액세스를 취소할 수 있습니다. 연체료/수수료 미지급 또는 트레이트 데이터를 부적절하게 사용하는 등의 이유로 피드에서 구매자를 삭제할 수 있습니다.

<!-- marketplace-deactiva4te-subscribers.xml -->

가입자를 취소하려면 다음 작업을 수행하십시오.

1. 위치 [!UICONTROL My Shared Data]구독자가 사용 중인 피드를 찾습니다.

   >[!NOTE]
   >
   >기한이 지난 계정이 있는 데이터 피드는 삼각형/느낌표 아이콘으로 플래그가 지정됩니다.

1. 다음에서 [!UICONTROL Subscribers] 열에서 해당 피드에 대한 구독자를 계산하는 파란색 숫자를 클릭합니다. 가입 세부 사항 페이지가 열립니다.
1. 이동 **[!UICONTROL Subscription]** 슬라이더 **[!UICONTROL Off]**. 확인 대화 상자 창이 열립니다.
1. 다음에서 [!UICONTROL Confirmation] pop, 클릭 **[!UICONTROL Yes]** 구독을 비활성화하려면 **[!UICONTROL Cancel]** 을 눌러 구독을 변경하지 않고 종료합니다.

### 가입자를 비활성화하면 어떻게 됩니까?

데이터 피드에 대한 액세스를 취소하면 데이터 구매자 계정의 모든 관리자 사용자에게 알림 이메일을 보냅니다. 이메일에는 해지된 트레이트가 나열된 첨부 파일이 포함됩니다. 이 목록은 구독자가 세그먼트 및 모델에서 비활성화된 트레이트를 찾고 제거하는 데 도움이 됩니다.

### 청구 및 피드 비활성화

데이터 피드에 대한 액세스 권한을 제거하면 피드를 비활성화한 시기에 따라 가입자는 이전 또는 현재 달의 요금을 물어야 합니다.

## 계획 유형 및 사용 사례 옵션 {#plan-types-use-cases}

<!-- c_feed_options.xml -->

다음 [!UICONTROL Use Case] 설정을 통해 판매자는 구매자가 데이터를 사용할 수 있는 방법을 제어할 수 있습니다.

### 세그먼트 및 겹치기

A **[!UICONTROL Segments and Overlap]** 사용 사례는 구매자가 의 트레이트 데이터를 비교할 수 있도록 해주는 계획을 만듭니다. [특성-특성 중복 보고서](../../../reporting/dynamic-reports/trait-trait-overlap-report.md#trait-to-trait-overlap-report). 또한 구매자는 세그먼트에 데이터를 추가하고 [세그먼트-트레이트](../../../reporting/dynamic-reports/segment-trait-overlap-report.md) 및 [세그먼트-세그먼트](../../../reporting/dynamic-reports/segment-segment-overlap-report.md) 보고서.

각 데이터 피드에는 최소 하나 이상이 포함되어야 합니다 [!UICONTROL Segments and Overlap] 사용 사례. 피드에 다음이 포함되지 않은 경우 구매자는 데이터 피드에서 다른 플랜에 가입할 수 없습니다. [!UICONTROL Segments and Overlap] 사용 사례(단독으로 또는 다른 사용 사례와 조합).

중복 비교는 구매자에게 도움이 될 수 있습니다.

* **대상자 도달 범위 확장:** 겹치는 정도가 낮으면 구매자가 이전에 보지 못한 사용자가 트레이트에 포함되어 있음을 나타냅니다. 그 결과 구매자는 이러한 트레이트가 새로운 사용자를 자신의 대상자 세그먼트에 추가하기를 원할 수 있습니다.
* **기존 대상 향상:** 겹치는 정도가 높으면 트레이트에 구매자가 이미 알고 있는 사용자와 유사한 사용자가 포함되어 있음을 알 수 있습니다. 따라서 구매자는 이러한 트레이트가 개발된 대상자를 타겟팅하고 점진적으로 개선하는 데 도움이 되기를 원할 수 있습니다.

이 사용 사례의 가격은 다음과 같습니다.

* 측정 단위: 정액 요금
* 가격: 무료 ($0.00)

### 모델링

A **[!UICONTROL Modeling]** 사용 사례는 구매자가 트레이트를 와 비교할 수 있도록 하는 계획을 생성합니다. [알고리즘 모델링](../../../features/algorithmic-models/understanding-models.md#understanding-models). 구매자는 모델 결과를 보고 데이터에서 자신과 유사한 전환 속성을 공유하는 새로운 대상을 찾습니다. 이 사용 사례의 가격은 다음과 같습니다.

* 측정 단위: 정액 요금
* 가격: 할인 또는 시장 요금 가격

### 활성화

An **[!UICONTROL Activation]** 사용 사례는 구매자가 로 데이터를 보낼 수 있도록 해줍니다 [대상](../../../features/destinations/destinations.md). 이 사용 사례를 사용하면 구매자는 중복 보고서나 알고리즘 모델의 데이터를 비교할 수 없습니다. 이 사용 사례의 가격은 다음과 같습니다.

* 측정 단위: [!DNL CPM]
* 가격: [!DNL CPM] 시장률

## 플랜 메모 {#plan-notes}

다음에서 **[!UICONTROL Additional Notes]** 필드, 피드의 각 데이터 플랜을 설명하는 데 약간의 시간이 소요됩니다. 간단하고 좋은 설명을 통해 구매자는 데이터 피드에서 각 플랜의 내용이나 목적을 이해할 수 있습니다. 구매자는 새 데이터 소스를 검색하거나 평가할 때 데이터 피드 및 계획 설명을 읽을 수 있습니다.

## 개인 데이터 피드 요청 관리 {#manage-private-requests}

구매자의 비공개 피드 요청 관리를 위한 공급자 워크플로우입니다.

구매자 요청을 검토, 승인 또는 거부하려면 [!UICONTROL My Shared Data] 및:

<!-- t_private_feed_workflows.xml -->

1. 개인 데이터 피드의 이름을 클릭합니다.
2. 클릭 **[!UICONTROL Access Requests]** 데이터 피드에 액세스하려는 모든 구매자를 리뷰합니다.
3. 다음에서 [!UICONTROL Allow Access] 각 요청 상자의 섹션에서 확인 표시를 클릭하여 요청을 승인하거나 X를 클릭하여 액세스를 거부합니다.
4. 확인 팝업에서 선택한 작업을 확인하거나 취소합니다.

## 데이터 제공업체 할인 {#discounts}

위치 [!UICONTROL Audience Marketplace]를 할인하면 개별 가입자에 대한 데이터 피드의 게시 가격을 절감할 수 있습니다. 구독 요청을 제출한 구독자 또는 데이터 피드에 대한 세부 정보를 요청한 구독자에게 할인을 제공할 수 있습니다. 할인 적용 대상 [!DNL CPM] 및 정액 피드. 신규 고객에게 구독 인센티브를 제공하거나 고객 충성도에 보답하고자 할 때 할인이 도움이 될 수 있습니다.
